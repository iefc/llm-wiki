# CLAUDE.md - LLM Wiki Schema

这是知识库的宪法。它告诉 LLM：Wiki 的结构是什么、Convention 有哪些、Ingest/Query/Lint 时遵循什么流程。

## 核心原则

1. **你写 Wiki，用户读 Wiki**。用户不写 Wiki（除非罕见的手动修正）
2. **Wiki 是持久、复合的 artifact**。不是每次查询重新生成，是随时间积累的结晶
3. **一次编译，持久持有**。新资料来了——更新现有页面 + 建链接 + 标注冲突，不是重新索引
4. **raw/ 不可变**。使用 `chmod -R a-w raw/` 强制文件系统级只读

## 页面格式规范

每个 Wiki 页面（MUST 包含）：

```markdown
---
title: {{页面标题}}
tags: [tag1, tag2, tag3]
created: {{YYYY-MM-DD}}
updated: {{YYYY-MM-DD}}
sources: [raw/源文件名.md]
---

# {{页面标题}}

## 定义

简洁的 one-liner 定义

## 核心要点

- 关键点 1
- 关键点 2

## 关联

- [[相关页面A]]
- [[相关页面B]]

## 来源

- raw/文件名（时间）

## 冲突标注（如有）

[CONFLICT]
- 观点A：...
- 观点B：...
```

文件名使用 kebab-case（如 `llm-wiki-vs-rag.md`）。

## frontmatter 必填字段

| 字段 | 说明 |
|------|------|
| `title` | 页面标题 |
| `tags` | 标签数组，用于分类和搜索 |
| `category` | 主分类目录（如 `aiops-与智能巡检`），对应 `wiki/` 下的子目录 |
| `created` | 创建日期 YYYY-MM-DD |
| `updated` | 更新日期 YYYY-MM-DD |
| `sources` | 关联的 raw/ 源文件 |

## 链接规范

- 使用 `[[页面名]]` 格式（Obsidian 双向链接）
- **跨目录引用**：同一子目录内直接用 `[[页面名]]`；跨子目录时用 `[[子目录/页面名]]`（如 `[[aiops-与智能巡检/OpenClaw]]`）
- 每个页面至少链接到 2 个其他页面
- 新建页面时同步更新 `index.md`

## 分类子目录规则

Wiki 文章按概念领域组织到 `wiki/` 下的子目录中。确定分类的优先级：

1. **主要主题**：文章核心讨论的概念领域 → 主分类
2. **关联领域**：文章涉及但非核心的领域 → 在关联段落中引用，不改变主分类
3. **新建分类**：当主题不属于现有 8 个分类时，创建新分类（中文命名，kebab-case 化）

现有 8 个分类：ai-工具与平台、监控与可观测性、容器与云原生、自动化运维、运维平台选型、aiops-与智能巡检、ai-agent、知识管理。

## 更新策略（Ingest 时）

1. 读取 `raw/` 中新增的文件
2. 与用户讨论关键要点（问答式交互），**确定主分类**（放在哪个子目录）
3. 搜索 `wiki/` 中是否已有相关页面
4. **已有页面** → 更新关联部分 + 补充核心要点 + 冲突则标注
5. **新主题** → 创建新页面到对应分类子目录 + 更新 `index.md`
6. 单次 ingest 可能涉及 10-15 个页面的更新
7. **涉及 10+ 页面时，先询问用户确认**
8. 完成后在 `log.md` 追加 entry

## Query 时

1. 读取 `index.md` 找到相关页面
2. 读取相关页面，综合生成回答
3. **有价值的回答 → 询问用户是否沉淀为 Wiki 页面**
4. 更新 `hot.md` 记录本次会话上下文

## Lint 时检查（8 类）

| 检查类型 | 说明 | 解决方式 |
|---------|------|---------|
| 孤立页面 | 无 inbound 链接 | 添加到相关页面或合并 |
| 死链 | 指向不存在页面 | 创建缺失页面或修正链接 |
| 矛盾检测 | 页面间冲突声明 | 添加 [CONFLICT] 标注 |
| 缺失页面 | 被提及但未创建的概念 | 创建新页面 |
| 未链接提及 | 提到但未建立链接的实体 | 添加 [[wikilink]] |
| 不完整元数据 | frontmatter 缺失字段 | 补全必要字段 |
| 空白段落 | 内容不完整的页面 | 补充内容或标记为草稿 |
| 过期索引 | index.md 与实际不一致 | 同步更新 index.md |

**LLM 主动建议**：新问题、新来源、值得深挖的方向。

## 冲突处理

```markdown
[CONFLICT]
- 旧观点：...（来自 raw/旧文件）
- 新观点：...（来自 raw/新文件）
- 保留两种观点，由用户判断
```

## 链接规范

- 使用 `[[页面名]]` 格式（Obsidian 双向链接）
- 每个页面至少链接到 2 个其他页面
- 新建页面时同步更新 `index.md`
